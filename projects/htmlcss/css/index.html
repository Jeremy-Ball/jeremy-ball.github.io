Designing the nav

I have marked borders around each element with colors. While using flexbox, I can easily align items center to each other. I also utilized static heights to keep my nav bar in the same position throughout every page I add on. The nav bar does not sit directly against the sides of the body. By keeping the width at 95% I can assure that the nav bar will always "float" within the same position inside the body. I use the same principal in the .nav-links container. "Floating" the navigation links within it. This will ensure that the nav link will grow easily with larger screen sizes.
The id #home is used for the icon image that will always take you back to the home page.

The body and image

The article is made into a flex box container to hold the pieces of the text within. An '.svg' image is used for the main graphic for its ability to resize without losing quality. Both of these containers will be easy to position above, below, or beside eachother. The article's items will be manipulated by using their width. By setting the width of the .article h1 and .article p to 98% I can assure that they will always sit above the social media links

The footer

The footer stays simple to keep to the minimalist design. Each 'p' tag will be easy to float and wrap as the sizes change using flexbox. I end the 'body' tag, then 'html' tag at the very end. My favorite part about this project is how simple and easy the tags are to organize. Pieces like the header, nav, and footer will be copy/pasted across every page on the site. This makes it easy and quick to update with very little code.
