Validating user input

This method was designed to specifically validate the input of users when they enter the month. It was designed to ensure that the user cannot enter anything but an integer between the numbers 01 (for January) and 12 (for December). Any other input will be denied with an appropriate prompt presented to inform the user why their input was not accepted.

Protecting against code injection

Some input requires letters, numbers, and other characters from the user. Code injection is when a hacker uses one of these input methods to type in code that could execute on the server holding the program. This can be negated by ensuring that the user input cannot be very long and checking ti against a function called .isalnum(). This ensures that the input cannot have any special characters.

Using error messages

Sometimes we can use error messages to help validate user input. This code checks if the user input is a number. If it isn'tx for example, if it's "q" to quit, then the program will throw an error. We can catch that error with a try/except statement. If an error code is thrown because of anything other than a number the program will then check if the input was "q". If it was not it will reprompt the user to enter a number.
