Mobile devices

300px and up

This is the first project that I have done mobile-first and I have found that the code is much cleaner and easier when sizing up. Over 70% od the CSS code used on this website is fit for 300px. Using flex-box and percentages the website can easily scale up relative to the device width. By using max-widths on the body I can assure that the website never scales too far. And since I fit elements into flexbox containers they will always stay within their relative boxes.

Tablets & Chromebooks

560px and up

I used a chromebook for most of my schooling and I always appreciated the work people would put in to make sure content could fit on it. At 560 and up the content stays centered. The only big thing I change are the width of the article and the flex-flow of the footer to row. The change required very little code.

Laptops and PC's

1000px and up

This is for larger screen sizes. I set a limit to the width of to body to 2000px so it will never stretch further than that. With the margins set to auto the body should sit in the middle of very large screens like TV's and projectors. The article and image are given a width of 45% so they will always sit side by side in the body and the width of the footer and nav is made smaller.
